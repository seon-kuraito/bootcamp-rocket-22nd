## 🚀 專案介紹
> 「致力於培養能在團隊中高效協作、善用 AI 與專業技術，並具交付能力的工程師。」

本專案為 [火箭隊培訓營](https://www.rocket-coding.com) 第 22 屆的 **「開放式任務系統」**。

將真實企業的需求量化為可操作的每週任務，以階段性目標與可交付成果為主軸，鍛鍊團隊協作與工程師思維。

此外，本專案亦作為學員的 **「公開成果紀錄」**。

所有 Issue、回饋與協作脈絡皆完整保存，讓每位學員的學習與成長得以被看見、查驗與追溯。

　

## 🧭 培訓方式
> 「每一份 Issue，都是一次真實的開發需求。」

本專案的培訓方式為「模擬真實工程團隊」的工作流。

以 GitHub 作為核心協作平台，讓學生在培訓期間學會：

- 使用 Issue 進行任務管理
- 在 Issue 中回報進度、提出問題與紀錄討論
- 撰寫清晰的 Comment 與溝通紀錄，提高協作效率
- 透過 PR 與 Code Review 精進程式碼品質

　

## 🧩 任務系統
> 「以任務為核心的學習循環，讓每週挑戰都貼近真實專案。」

所有任務都有明確輸出與可複製的流程，讓學習成果能被驗證與累積：

- 以 GitHub Issue 形式指派，每週日 23:00-24:00 更新當週任務
- 學員於 Issue 提交成果、反思與進度回報
- 教練在 Issue thread 給予回饋

任務難度會隨週次逐步提升，並搭配跨週循環的專案規劃。

👉 [查看所有任務](https://github.com/seon-kuraito/bootcamp-rocket-22nd/issues)

## 🌟 教練
> 「教練提供方向與回饋，交付與成長由你主導。」

火箭隊的教練團隊由具備實務經驗的工程師與設計師組成。

教練並非單純給予答案，而是在每一次任務中協助學員釐清思考、提升表達、並透過系統化回饋建立可持續成長的能力。
